Cloudflare Worker setup
The dashboard reads live data from Linear via a Cloudflare Worker you deploy in minutes. It's free — Cloudflare's free plan includes 100,000 requests/day.
1
Create a free Cloudflare account
Go to workers.cloudflare.com and sign up for free. No credit card required.
2
Create a new Worker
In the Cloudflare dashboard → Workers & Pages → Create → Worker. Name it anything (e.g. cqa-linear-proxy).
3
Paste the Worker code
Click "Edit code", replace everything with the code from worker.js (provided alongside this dashboard file). Deploy it.
4
Add your Linear API key
In your Worker → Settings → Variables → Add variable:
Name: LINEAR_API_KEY  Value: your Linear API key (Settings → API in Linear).
5
Paste your Worker URL below
Your Worker URL looks like https://cqa-linear-proxy.yourname.workers.dev
Worker URL
Paste your Cloudflare Worker URL. This is saved in your browser.
Work classification rules
The dashboard uses these rules to classify each Linear issue into Client Delivery, POC/Prospect, or Product Testing. Rules are matched against the issue title, labels, and project name. First matching rule wins.
Client list override
The dashboard auto-detects clients from your Linear projects and labels. You can also manually add client names here — useful when client names appear in issue titles rather than as structured fields.
The dashboard will match these names against issue titles, labels, and project names to identify client work.
